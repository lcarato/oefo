DATA SOURCES

Development Finance Institutions (DFIs):
1. IFC (International Finance Corporation)
   - Target: disclosures.ifc.org
   - Focus: Energy sector project disclosures
   - Document types: ESRS, SII, PAD PDFs
   - Methods: search_projects, scrape_project_page, download_project_documents

2. EBRD (European Bank for Reconstruction and Development)
   - Target: ebrd.com/projects
   - Focus: Energy projects with CSV master list
   - Unique: Downloads official project CSV then individual PSDs
   - Methods: download_master_csv, filter_energy_projects, fallback scraping

3. GCF (Green Climate Fund)
   - Target: greenclimate.fund/projects
   - Focus: Climate mitigation/adaptation energy projects
   - Document types: 50+ page funding proposals
   - Methods: list_projects, scrape_project_page, download_funding_proposal

Corporate Filings:
4. SEC EDGAR (US Securities and Exchange Commission)
   - Target: efts.sec.gov (Full-Text API) + data.sec.gov (XBRL API)
   - Focus: US company filings (10-K, 10-Q, 8-K, S-1)
   - Financial metrics: WACC, cost of capital, capital structure
   - Rate limit: 0.1s (SEC requirement: max 10 requests/second)
   - APIs: Full-text search + XBRL structured data
   - Methods: search_by_keyword, get_xbrl_data, download_filing

Regulatory Agencies:
5. ANEEL (Brazil - Agência Nacional de Energia Elétrica)
   - Target: aneel.gov.br/regulacao-tarifaria
   - Language: Portuguese
   - Focus: Tariff reviews with CAPM decomposition
   - Value: Publishes complete WACC breakdown (very high value)
   - Metrics: Risk-free rate, equity risk premium, beta, cost of debt

6. AER (Australia - Australian Energy Regulator)
   - Target: aer.gov.au/networks-pipelines/.../rate-of-return
   - Language: English
   - Focus: Rate of return determinations
   - Value: Binding 5-year determinations with Excel models
   - Metrics: CAPM models, cost of equity, cost of debt

7. Ofgem (UK - Office of Gas and Electricity Markets)
   - Target: ofgem.gov.uk/price control pages
   - Language: English
   - Focus: RIIO price control decisions
   - Metrics: WACC, cost of equity, cost of debt
   - Periods: 8-year regulatory cycles with annual updates

8. FERC (US - Federal Energy Regulatory Commission)
   - Target: ferc.gov + elibrary.ferc.gov
   - Language: English
   - Focus: Rate cases and project orders
   - Metrics: Return on equity (ROE), capital structure
   - Documents: Order 888/889, certificate applications, licensing

================================================================================
KEY CAPABILITIES
================================================================================

RATE LIMITING
- Default: 1.0 second between requests
- SEC EDGAR: 0.1 seconds (regulatory compliance)
- Configurable per scraper instance
- Uses time.sleep() for precise control

RETRY LOGIC
- 3 automatic retries with exponential backoff
- Backoff delays: 1s, 2s, 4s
- Handles: 429 (rate limited), 5xx (server errors)
- Respects HTTP timeouts (30 seconds default)

CONTENT DEDUPLICATION
- SHA-256 hashing of downloaded files
- Prevents re-downloading identical documents
- Hash stored in RawDocument metadata
- Maintains document_store set per scraper

METADATA TRACKING
- RawDocument Pydantic model with 15+ fields
- Tracks: source URL, hash, file path, download date
- Includes: MIME type, file size, document title
- Audit trail for all downloaded content

ERROR HANDLING
- HTTP errors: automatic retries
- Network errors: logged, continues processing
- Parsing errors: fallback strategies
- File I/O: graceful degradation
- Comprehensive error messages with context

LOGGING
- Hierarchical loggers by module
- Levels: DEBUG (verbose), INFO (progress), WARNING (issues), ERROR (failures)
- Logs HTTP requests, downloads, parsing, registration
- Configurable at application level

PERFORMANCE CHARACTERISTICS
================================================================================

Typical Rates:
- 1 document per second (with rate limiting)
- 20-50 documents per hour (realistic)
- Large PDFs (50+ pages): 5-30 seconds download
- Content hash: ~100ms per file
- BeautifulSoup parsing: ~50ms per page

Memory Usage:

- 50-100 documents session: 50-100MB total
- Streaming downloads (no full buffer)

Concurrency:
- Single-threaded design (respects rate limits)
- Can parallelize with ThreadPoolExecutor (see ARCHITECTURE.md)
- Adjust max_workers based on rate limits
